现在我们就可以实现“Git远程连接GitHub”
前提：
 1. 安装Git
 2. 注册Github账号

三、远程连接（只做一次）
   1. 设置用户名:git config --global user.name '自己的用户名'
          ===git config --global user.name 'liuliu408'
   2. 设置用户名邮箱:git config --global user.email '自己的邮箱'
          ===git config --global user.email 'liuliu408@163.com'

四、使用
   1.把远程的仓库下载下来（创建一个本地仓库的克隆版本）
    如：git clone git@github.com:liuliu408/tracking.git
   接下来就可以实现下面单位功能
   2.可以删除文件后更新远程仓库
   3.可以添加文件的到远程仓库
   4.可以上传本地仓库文件到远程仓库！

Ubuntu下git本地仓库与远程仓库的同步
一、将远程仓库同步到本地
1. mkdir liuq //创建一个文件夹liuq
2. cd liuq    //进入liuq文件夹
3. git init   //使他成为一个仓库
4. git remote add origin *****   // ***为你的远程practice仓库的“clone ordownload”将网址复制
   如：git remote add origin git@github.com:liuliu408/tracking.git
   这个命令只用输一次，以后就不用再输入了，直接git add，git commit，git pull，git push。
5. git pull  git@github.com:liuliu408/tracking.git
   备注：pull把我们远处仓库的文件下下来

   要更新你的本地仓库至最新改动，执行：git pull（git pull #将远程仓库里面的项目拉下来）

6.  把远程的仓库下载下来（创建一个本地仓库的克隆版本）
    如：git clone git@github.com:liuliu408/tracking.git

7. 假如你想要丢弃你所有的本地改动与提交，可以到服务器上获取最新的版本并将你本地主分支指向到它!!
	git fetch origin
	git reset --hard origin/master


二、将本地仓库文件的修改提交到远程仓库
1. cd liuq        //进入liuq文件夹
2. git add test   //添加到暂存区，把本地仓库的文件上传到缓存
   备注：现在修改的文件加入到了暂存区，但是还没有提交到远程仓库
3. git commit -m 'add a file'  // -m 后的内容是对此次修改的说明
   备注：将文件从暂存区提交到仓库,即把第一步上传到缓存的东西上传到本地仓库 ，现在你的改动已经提交到了 HEAD，但是还没到你的远端仓库。
4. git push -u origin master   //这样就提交成功了！
   执行如下命令以将这些改动提交到远端仓库，可以把 master 换成你想要推送的任何分支。

三、删除仓库里面的文件，下面举例删除test文件
 (1） 删除文件：rm -rf test
 (2)  从Git中删除文件：git rm test
 (3)  提交操作：git commit -m ‘删除test’
 (4)  执行如下命令以将这些改动提交到远端仓库：git push -u origin 分支名（分支名通常为master）
      git push -u origin master

一次删除所有的：
git rm *    			 -----删除仓库里面的所有文件
git commit -m '此处添加删除注释'  -----提交操作，将文件从暂存区提交到仓库	 
git push -u origin master        -----提交到远端仓库


git rm -r --cached target          # 删除target文件夹

从远程分支获取最新版本git fetch与git pull
1，git fetch。相当于是从远程获取最新版本到本地，不会自动merge（合并）。
2，git pull。相当于从远程获取最新版本兵合并到本地。
   git pull origin master就相当于git fetch 和git merge。


    使用git在本地创建一个项目的过程

    $ makdir ~/hello-world    //创建一个项目hello-world
    $ cd ~/hello-world       //打开这个项目
    $ git init               //初始化 
    $ touch README
    $ git add README        //更新README文件
    $ git commit -m 'first commit'     //提交更新，并注释信息“first commit”
    $ git remote add origin git@github.com:liuliu408/tracking.git   //连接远程github项目  
    $ git push -u origin master     //将本地项目更新到github项目上去、

常用问题：
1. git添加公钥后报错的解决办法:
liuq@liuq-Z270P-D3:~/liuq_github$ git push -u origin master
sign_and_send_pubkey: signing failed: agent refused operation
Permission denied (publickey).
fatal: Could not read from remote repository.

====这个时候我们只要执行下:
	eval "$(ssh-agent -s)"
	ssh-add
　　就可以了!


2.   如果输入$ git push origin master
    提示出错信息：error:failed to push som refs to .......
    解决办法如下：
    1、先输入$ git pull origin master //先把远程服务器github上面的文件拉下来
    2、再输入$ git push origin master
